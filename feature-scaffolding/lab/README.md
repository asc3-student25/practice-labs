# Notification Dispatcher — Scaffolding Lab

This repository is deliberately almost-empty. It contains only this README, a minimal `pyproject.toml`, and a feature brief. You will use Copilot to scaffold the microservice described in the brief, then audit what the agent produced against your own expectations.

The lab is not about writing code that runs. It is about driving an agent to produce *structure* and evaluating the result critically.

## Files You Will Create

- `BASELINE.md` — your hand-sketched directory and interface expectations (before prompting Copilot)
- Generated source directories and stub files from the scaffolding runs
- `AUDIT.md` — your critique of the generated scaffolding

## What Is Already Here

- `FEATURE_BRIEF.md` — the microservice specification
- `BASELINE_TEMPLATE.md` — a template for `BASELINE.md`
- `pyproject.toml` — a minimal project configuration with no dependencies and no structural hints
- `solutions/` — two reference scaffolds (layered and hexagonal) plus `EVALUATION_CHECKLIST.md`

## Important

Open the reference scaffolds *after* you have evaluated your own run. Reading them first biases your baseline and defeats the audit exercise.
