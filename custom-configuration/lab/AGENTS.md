# Agent Behavioral Guidance

These rules apply to any autonomous agent session (Agent Mode, the Copilot coding agent, or Copilot CLI) working in this repository. They are in addition to the Chat-level instructions under `.github/`.

## Hard constraints

- **Never edit existing files under `migrations/`.** Committed migrations are append-only. If a schema change is needed, create a new numbered file `migrations/NNNN_<short_description>.py`.
- **Never commit or modify files under `build/`, `generated/`, or `node_modules/`.** These are produced by build tooling and must not be edited by hand.
- **Never modify `package-lock.json` by hand.** It is regenerated by the package manager.

## Before acting

- When a task touches the schema or the User model, state whether a new migration is needed before proposing code changes. If yes, show the new migration filename first.
- When a task would require a new dependency, state the dependency and wait for approval before editing `requirements.txt`.

## Scope

- Stay inside the directory implied by the task unless the change genuinely requires a cross-directory update (for example, a new endpoint that requires a new test).
- If a change requires touching files in more than two top-level directories, pause and confirm the full list with the requester before proceeding.

## Reporting

- After completing any task that added or modified routes, print a short summary listing the endpoints added and the tests that cover them.
- After any task that modifies a model, print the set of files affected (model, store, serializer, tests, new migration).
